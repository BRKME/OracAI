# CYCLE LADDER POLICY v1.0 — зона цикла → деньги

Зафиксировано 2026-06-10. Менять только версионным апдейтом политики, никогда
ad hoc — особенно не в момент страха или эйфории (политика существует именно
для этих моментов).

## Зачем

Аудиты показали одновременно две вещи: дневная классификация режима не
добавляет почти ничего к однострочному SMA200-правилу, а медленные
валюационные крайности (MVRV) надёжно различимы 13 лет подряд. Деньги цикла —
не в точности детекции «сегодня», а в нескольких больших заранее принятых
решениях: покупать сильнее ниже себестоимости рынка, фиксировать в перегреве,
разворачивать резерв по объективному триггеру, и не импровизировать в
неоднозначной середине.

## Таблица (ladder-v1.0)

| Зона | MVRV | Недельный DCA | Фиксация | Комментарий |
|---|---|---|---|---|
| ACCUMULATION | < 1.0 | **×2.0** | — | Ниже себестоимости рынка. Покупать страшно — поэтому решение принято заранее |
| NEUTRAL | 1.0–1.5 | ×1.5 | — | **Guard:** при `STRUCTURAL_BEAR_RISK` множитель удержан на ×1.0 (урок 2022) |
| EXPANSION | 1.5–2.2 | ×1.0 | — | База. Здоровая середина, ничего не выжимаем |
| DISTRIBUTION | 2.2–3.0 | ×0.5 | **25% стэка** при входе в зону | Этот цикл прошли без фиксации — это и стоило −50% |
| EUPHORIA | ≥ 3.0 | **×0** | **50%** | Покупки стоп |

**Re-risk резерва:** ≥10 дней подряд выше SMA200 → `re_risk: true`
(тот же объективный триггер, что Phase 4 recovery override).

Зона приходит из `cycle_context` (MVRV первичен, Mayer как price-only fallback,
трёхуровневый каскад источников MVRV).

## Контракт: state/cycle_ladder.json

Публикуется ежедневно воркфлоу `cycle_layer_log` (~02:45 UTC). Потребитель —
hl_weekly_planner (субботний DCA-прогон), fetch по конвенции BRKME:

```
https://raw.githubusercontent.com/BRKME/OracAI/main/state/cycle_ladder.json
```

```json
{
  "policy_version": "ladder-v1.0",
  "zone": "NEUTRAL",
  "drawdown_call": "AMBIGUOUS",
  "mvrv": 1.15,
  "dca_multiplier": 1.5,
  "fixation_fraction": 0.0,
  "re_risk": false,
  "days_above_sma200": 0,
  "rationale": "...",
  "date_utc": "2026-06-10",
  "price": 61380.0
}
```

Правила потребления на стороне planner:
- `dca_multiplier` умножает базовую недельную ставку DCA.
- `fixation_fraction > 0` — разовое действие **при смене зоны** (planner хранит
  последнюю виденную зону; фиксация не повторяется каждую неделю внутри зоны).
- `re_risk: true` — сигнал разворачивать накопленный резерв (ступенями, не разом).
- `date_utc` старше 3 дней ИЛИ файл недоступен → fail-safe: множитель ×1.0,
  без фиксаций (контракт деградирует в «обычный DCA», не в крайность).

## Текущее состояние (на момент фиксации политики)

BTC −50.8% от ATH, MVRV ~1.15, Mayer 0.79 → NEUTRAL / AMBIGUOUS → **×1.5**,
re-risk нет. Лестница говорит: усиленное накопление уже уместно, агрессия ×2.0
ждёт MVRV < 1.0, разворот резерва ждёт 10 дней над SMA200. Если придёт
`STRUCTURAL_BEAR_RISK` — откат на ×1.0 автоматически.

## Что НЕ делает лестница

Не предсказывает направление, не торгует сама, не отменяет дисциплину
замороженного движка (она читает его выход, не меняет его), не действует в
середине цикла — там она равна обычному DCA, и это правильный ответ.

## v1.1 (2026-06-10): событийные сигналы в % капитала

По решению оператора: никаких сумм — сигналы в процентах капитала, разово при
смене зоны. Множитель v1 остаётся в контракте для совместимости; основной
механизм теперь `signal`:

| Событие | Сигнал |
|---|---|
| Вход в NEUTRAL | **BUY 30% капитала** (×0.5 при STRUCTURAL_BEAR_RISK — 15%, остальное ждёт ACCUMULATION) |
| Вход в ACCUMULATION (MVRV<1.0) | **BUY 30% капитала** |
| ≥10д выше SMA200 (раз за цикл) | **BUY остаток** (1.0 − потрачено) |
| Вход в DISTRIBUTION | **SELL 25% стэка** |
| Вход в EUPHORIA | **SELL 50% стэка** + сброс buy-бюджета на следующий цикл |
| Иначе | HOLD, молчание |

Бюджет покупок сходится к 100% капитала за полный спуск; фактически
потраченная доля хранится в `state/ladder_state.json` (коммитится воркфлоу —
это память одноразовости). BUY/SELL уходят в Telegram немедленно; HOLD не шлётся.

Поле контракта:
```json
"signal": {"policy_version": "ladder-v1.1", "action": "BUY",
           "fraction_of_capital": 0.30, "fraction_of_stack": 0.0,
           "trigger": "entered_NEUTRAL", "rationale": "..."}
```

Первый живой прогон после деплоя пришлёт «BUY 30%» — мы уже в NEUTRAL, и эта
ступень лестницы по политике должна быть развёрнута.
